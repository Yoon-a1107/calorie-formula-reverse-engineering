##  한 줄 요약

웨어러블 센서 데이터(심박수·체온·운동시간 등)로 칼로리 소모량을 예측하는 ML 프로젝트.

단순히 좋은 모델을 찾는 것이 아니라, **"데이터가 어떤 공식으로 만들어졌는가"** 를 역공학으로 규명하여 이론적 한계치에 근접한 정밀도를 달성했다.

>  **배운 것**: 복잡한 피처를 많이 만드는 것이 항상 정답이 아니다.
> 문제를 올바르게 정의하고, 오차 패턴을 다양한 시각으로 바라볼 때 단순한 선형 공식이 최신 모델보다 좋은 지표를 만들 수 있다.

---

## 문제 정의 - 왜 이 접근법인가

처음에는 XGBoost, LightGBM, CatBoost 등 트리 계열 모델로 접근했다.
그런데 아무리 튜닝해도 RMSE가 **1.0 아래로 내려가지 않았다.**

원인을 추적하자 예상치 못한 사실이 드러났다.

> **"트리 모델은 0.5 단위의 반올림 경계선을 맞추려고 리프 노드를 끝없이 뻗어나가다가 전체 규칙을 망가뜨리고 있었다."**

정답은 곡선이 아니라 **직선**이었다.
칼로리 소모량이 본질적으로 선형 관계를 가진다는 가설을 세우고, 생리학 문헌을 탐색하여 데이터 생성에 사용된 공식을 역공학으로 찾아냈다.

## 🔍 핵심 발견 3가지

### 1. Keytel 공식 역공학 (Reverse Engineering)

데이터가 생리학적 에너지 소비 추정 공식인 **Keytel 공식**을 기반으로 생성되었음을 규명.
성별에 따라 두 가지 선형 공식이 적용된다 (단위: kcal/min):

**남성:**
$$EE = (-55.0969 + 0.6309 \times HR + 0.1988 \times W + 0.2017 \times A) / 4.184$$

**여성:**
$$EE = (-20.4022 + 0.4472 \times HR - 0.1263 \times W + 0.074 \times A) / 4.184$$

> 📄 참고 논문: Keytel LR et al. (2005). *Prediction of energy expenditure from heart rate monitoring during submaximal exercise.* Journal of Sports Sciences, 23(3), 289–297.

이 공식을 **Ridge 회귀의 베이스 모델로 채택**, 복잡한 트리 모델 대신 선형 규제 모델이 훨씬 강력한 성능을 냈다.

---

### 2. 21명의 빌런 — Rounding Paradox 규명
 
전체 7,500건을 예측했을 때, **단 21건에서만 ±1.0의 고정 오차**가 반복적으로 발생했다.
 
패턴을 분석하자 이 21명 **전원이 체온 104°F(40°C) 이상**이었다.
 
처음엔 이 구간을 이상치로 단순 제거하려 했다. 그러나 격렬한 운동 시 근육 발열로 인해 심부 체온이 40°C 이상까지 올라가는 것은 생리학적으로 보고된 범위 안이다. **데이터 오류가 아니라 다양성(Variance)** 으로 판단하고 그대로 유지했다.
 
그렇다면 왜 하필 이 구간에서만 오차가 날까?
 
체온이 높아질수록 Keytel 공식 내 체온 관련 항의 값이 커지고, 그만큼 **부동소수점 연산 오차가 반올림 경계(0.5)를 넘기 쉬운 조건**이 된다. 여기에 데이터 생성 시 사용된 변환 상수(4.18403)와 우리가 적용한 상수(4.184) 사이의 미세한 차이가 맞물리면서 반올림 결과가 뒤집혔다.
 
| 질문 | 답 |
|------|------|
| 왜 21명인가 | 체온 40°C 이상 구간에서 부동소수점 오차가 반올림 경계를 넘는 조건이 충족됨 |
| 왜 오차가 정확히 ±1.0인가 | 반올림 특성상 경계선에서 오차는 0 또는 1로만 발생 |
| 40°C가 원인인가 | 원인이 아니라 조건. 이 구간에서 연산 오차가 경계를 넘기 쉬운 구조였던 것 |
 
> **결론:** 모델의 한계가 아니다. 변환 상수의 미세한 차이(4.184 vs 4.18403)가 고체온 구간에서 반올림 경계를 넘기면서 발생한 **연산 정밀도 문제(Rounding Paradox)** 다. 이론적으로 이 21건을 제외하면 나머지 99.72%는 오차 0이다.

### 3. 하이브리드 잔차 학습 아키텍처 설계

공식만으로는 해결되지 않는 21건의 잔차(Residual)를 처리하기 위해 2단계 파이프라인을 설계했다.

```
[입력 데이터]
     ↓
[Stage 1] Keytel 공식 기반 Ridge → 기본 예측값 (선형)
     ↓
[Stage 2] CatBoost → 잔차(오차) 예측 및 보정
     ↓
[최종 예측값 반올림]
     ↓
검증 RMSE 0.0529 (이론적 한계치)
```

## ⚙️ 피처 엔지니어링 주요 변수

| 변수명 | 설명 | 생리학적 근거 |
|--------|------|-------------|
| `Metabolic_Factor` | 체온 변화에 따른 대사 속도 지수 (1.13의 거듭제곱) | 체온 1°C 상승 시 대사율 약 13% 증가 |
| `EPOC_Factor` | 고온 + 고강도 운동 시너지 지표 | 운동 후 초과산소소모량(EPOC) 반영 |
| `Thermal_Strain` | 체온 × 운동시간 복합 열부하 지표 | 열 스트레스 누적 효과 |
| `HRR_Ratio` | 심박수 예비력 대비 현재 강도 비율 | Karvonen 공식 기반 |
| `BMR` | 성별 특화 기초대사량 | Mifflin-St Jeor 공식 |

총 20개 이상의 생리학 기반 파생 변수를 설계한 후, CatBoost Feature Importance와 SelectKBest로 핵심 피처를 선별했다.

>  **코치 피드백**: 파생 변수를 많이 만들수록 좋아진다는 생각에서 벗어나, 다중공선성과 모델 복잡도의 관계를 직접 경험으로 이해한 점이 인상적이었다. 피처를 많이 만드는 게 항상 좋은 결과로 이어지지 않고, 모델이 데이터를 암기하기 시작하면 오히려 성능이 떨어질 수 있다.

##  회고 — 상위 3팀 방식 분석

같은 데이터셋을 두고 세 팀이 완전히 다른 사고 경로로 접근했다.
각 팀의 접근을 비교하면 데이터 분석에서 어떤 사고방식이 문제를 풀어내는지 확인할 수 있었다.

---

### 문제 재정의 (그라운드 원, RMSE 0.000) 

> **"예측(Prediction)을 넘어 복원(Restoration)으로"**

가장 인상적이었던 부분은 **문제 자체를 재정의한 프레임 전환**이다.
XGBoost/LGBM이 RMSE 0.003에서 멈추는 현상을 보고, 모델 성능 부족이 아니라 **"통계적 확률에 의존하는 머신러닝의 태생적 한계"** 로 진단했다.
데이터 안에 통계를 넘어서는 **절대적 규칙이 존재할 가능성**을 포착한 것이 전환점이었다.

이후 KNN(k=1)을 단순 분류기가 아닌 **'데이터 기억 및 복원 엔진'** 으로 재정의하고, Kaggle Pre-trained Data를 지식 증류(Knowledge Distillation)해 모델 가중치에 박제하는 아키텍처를 설계했다.
Keytel 공식을 단순 활용에 그치지 않고, 데이터 기반으로 계수를 검증하고 단위 변환 정밀도(0.45359237)까지 확인한 과정이 0점을 가능하게 한 결정타였다.

---

### 오차 패턴 기반 귀납적 추론 (죽음의 다2어트조, RMSE 0.000)

> **"모델 실험 → 구조 가설 → 변수 축소 → 계수 패턴 확인 → 단위/반올림 규칙 정리"**

RMSE 0.11에서 정체되었을 때 모델을 복잡하게 만드는 대신, **오차가 ±1로 반복되는 패턴**을 보고 생성 규칙을 의심한 전환이 인상적이었다.
`Calories / Exercise_Duration` 변환으로 운동시간 효과를 분리해 변수 구조를 파악하고, 성별을 나눴을 때 RMSE가 6.83 → 0.29로 급락한 것도 데이터를 제대로 이해한 결과였다.
특히 Lasso/OLS를 **최종 모델이 아닌 구조를 이해하기 위한 도구**로 사용한 점이 목적이 명확해서 좋았다.
최종적으로 체중 항의 1/2.2 스케일 차이와 `int_kg + rint` 조합을 정렬하며 0점을 달성했다.

---

### 우리 팀의 도메인 기반 피처 설계 + 잔차 학습 회고

우리 팀도 Keytel 공식 역공학에는 성공했다.
그러나 **0점을 못 낸 이유를 짚어보면 두 가지**다.

**① 단위 변환의 정밀도를 끝까지 추적하지 못했다**
1등 팀은 체중 변환 상수로 `0.45359237`을 찾아냈지만, 우리는 `4.184 vs 4.18403` 수준의 엡실론 탐색에서 멈췄다. 체중 단위(lb → kg) 변환 정밀도까지 독립적으로 추적하는 것이 부족했다.

**② 오차 패턴 분석의 방향이 달랐다**
2등 팀은 오차 패턴에서 바로 "생성 규칙 문제"를 의심하고 귀납적으로 공식을 도출했지만, 우리 팀은 21건의 이상치를 생리학적 노이즈로 해석하는 방향으로 먼저 접근했다. 같은 이상치를 두고도 "왜 이 패턴인가"를 다른 방식으로 접근하지 못해 아쉽다.
